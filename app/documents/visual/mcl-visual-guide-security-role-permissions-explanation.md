# Role Permissions Explanation

The role permissions table explains how various permissions impact user capabilities. When 'Tasks can be assigned' is checked, users can receive tasks; unchecked, they cannot. 'Creation of tasks' permits manual task creation under Tasks → New Task when activated. Sensor-related menus and monitoring data access depend on the 'Sensors' and 'Monitoring data' permissions, respectively. Enabled 'Break lists' grant access to Break list modules, while checking 'Analytics email' sends weekly summaries of checklist performances. Only users with admin rights can edit permissions using the pencil icon.
